핵심 경고

모르는 사람이 제공한 솔라나 봇, HFT 봇, 스나이퍼 봇, 번들러, 볼륨 봇, 차익거래 봇, 지갑 관리툴, 텔레그램 봇, 웹 대시보드에 프라이빗키나 시드구문을 넣으면 안 됩니다. 한 번 노출된 키는 다시 안전해지지 않습니다.

개발자가 VPS 루트 권한을 가졌다면 코드뿐 아니라 서버 내부에도 흔적이 남을 수 있습니다. SSH 키, PM2 프로세스, systemd 서비스, crontab, 환경변수, 백업 파일, 로그, 웹훅, 숨겨진 시작 스크립트까지 확인해야 합니다.

이 보고서는 법원 판결이나 수사 결과가 아니라, 사용자 신고와 공개 단서를 기반으로 한 보안 경고입니다. 그러나 지갑과 서버 보안에서는 의심되는 키와 접근 권한을 즉시 폐기하는 것이 안전합니다.

피해야 할 위험 행동

실제 자금이 들어 있는 지갑의 프라이빗키를 .env 파일, 텔레그램 봇, 개발자 대시보드, 원격 VPS, 비공개 ZIP 파일, 컴파일된 실행 파일에 넣지 마세요. 테스트가 필요하면 반드시 소액 테스트 전용 지갑만 사용해야 합니다.

거래소 API 키, KuCoin API 키, RPC 키, Helius 키, Yellowstone gRPC 토큰, Jito 키, GitHub 토큰도 모두 민감 정보입니다. 직접 출금 권한이 없더라도 전략 노출, 서버 장악, 추가 공격에 사용될 수 있습니다.
