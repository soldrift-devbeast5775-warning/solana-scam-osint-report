What This Report Covers

The core identifier tracked here is Soldrift, with the Telegram handle @devbeast5775 and related public traces submitted by the complainant. The report is intended for Solana traders, token launch teams, server owners, and developers who may encounter wallet tools, sniper bots, HFT bots, bundlers, arbitrage dashboards, or Telegram-distributed trading software promoted under similar identifiers.

The complainant reports that access to a Solana trading / launch environment was provided during a development engagement and that the resulting codebase and server required a security review for private-key exposure, wallet-drain risk, malicious backdoor logic, outbound logging, and unauthorized developer access. This page records that report as a public safety warning, not as a court judgment.

The warning is intentionally broad because private-key tools can place funds at risk even when the visible interface appears professional. A working dashboard, a large GitHub profile, or a Telegram demo does not independently prove that wallet software is safe.

Risk Summary

The highest-risk scenario is a user pasting a funded Solana private key, seed phrase, RPC key, gRPC token, exchange API key, or VPS credential into an unknown tool. Once a private key is exposed to a third-party dashboard, the safest assumption is that the wallet must be retired.

Server access also creates risk. A developer with root access can add SSH keys, change systemd services, create PM2 processes, modify environment files, add crontab jobs, hide outbound webhooks, alter dependencies, or insert wallet export logic that is difficult for a non-technical buyer to detect.

The report does not claim that every tool, account, or wallet using a similar name is unsafe. It warns that this identifier cluster has been reported in connection with high-risk Solana wallet tooling and should be independently verified before any funds or credentials are exposed.

Immediate Safety Steps

Do not reuse any private key that was pasted into the tool, server, Telegram bot, dashboard, .env file, or deployment script. Move remaining funds to fresh wallets generated on a trusted local device. Rotate RPC keys, gRPC tokens, exchange API keys, GitHub tokens, server passwords, SSH keys, and any webhook secrets.

Inspect the VPS for unknown users, authorized_keys entries, PM2 processes, systemd units, crontab entries, startup scripts, modified package scripts, suspicious dependencies, outbound fetch calls, dynamic execution, base64 blobs, encrypted payloads, wallet export logic, Telegram webhooks, Discord webhooks, and remote logging.

Preserve public evidence with UTC timestamps before deleting anything. Useful evidence includes repository URLs, commit history, Telegram handles, screenshots, transaction hashes, public wallet addresses, archive snapshots, deployment logs, and server command output that does not expose secrets.
